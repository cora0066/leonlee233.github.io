一个好的开始

写在前头

好久不见了各位，这里还是洋洋，我又一次回来了
沉寂了一年的博客又被我捡起来了
这过去的一年真的发生了好多事情啊
我和我前对象分手
被真菌感染折磨，经历复发，手术
在学校恍恍惚惚就过到了暑假，在学校学习了一整个暑假
9月志愿填报，去医院手术，包皮手术术后钻心疼，真菌感染从此再也没复发过
临近考试了因为压力大把南昌的自助吃了个遍
疫情一夜之间消失，人心一夜之间惶惶
好在我聪明，提前买了一整瓶布洛芬，陆陆续续少说分给了十个人
黎明前的平静最终在我打碎了水银温度计和整层到处都是阳性感染者的之下被打破了
于是我和室友出去住了，以此来逃脱病毒的蔓延，可最终还是没躲过

头一回发烧到头昏脑涨，喉咙针扎般痛苦，如实填写健康信息而被分到了小阳人考场
艰难的在手机上敲下最终还是倒在黎明之前
考场上不绝于耳的咳嗽和试卷的沙沙声伴随着交卷铃声戛然而止，仿佛一切都结束了
寒假在家摆了两个月，见到了认识一年多的现对象，当时他还很胖很大一只
回学校等待考研成绩，还没等到考研成绩，先被四级吓了一大跳
没想到意外的考了三年唯一过的一次的四级成绩只有426，并且192分都在听力上
一时间不知是幸运的还是不幸，期盼着，恐惧着
期盼我的研究生考试也能像四级一样乘着狗屎运顺利通过
又害怕没有听力的研究生考试也会像这次四级一样一头栽在泥泞里
结果出来了，分数还算理想，开始着手复习复试

在南昌见了我的宝，第二次我们交谈甚欢，在星巴克点了人生中的第一杯星冰乐
不好喝，太甜腻了，不过每每想起那个场景，我都很爱那种感觉，大概是因为他吧
我们开始分享生活，在我去桂林之前的那个晚上，看到他发的“我喜欢你”四个字
感觉一切疲惫都消失了，在洗了个澡之后，我敲下了“我也喜欢你”
我们回过头来想那段时光的时候，我们俩都像头一回谈恋爱一般，
冲动，不知所措，迷迷糊糊，就像是小朋友谈恋爱一般，“我喜欢你”“那我们就在一起吧”
但是好在时间告诉我们，我们选对了

满怀着信心去复试，抱着未来三年要在这生活的美好想象，我踏上了去桂林的高铁
可这次的体验并不好，糟糕的城建，狭窄的道路，糟糕的旅店，答得稀烂的复试
在复试一结束我就改签最早的火车离开了这里，心想着找一个水一点的导师，研二就去大城市打工
找了高中的伙伴去湖南玩了两天，吃了笨萝卜，喝了茶颜悦色，调整好心情回到了学校
等复试成绩的过程是最煎熬的，做最坏的打算已经来不及了，
技术不行找不到工作，也不愿再考研一年浪费给家庭压力，
害怕功亏一篑，普通家庭的孩子就是这样，似乎只要做错一个选择，生活就会摇摇欲坠
仔细想想，有的时候只是害怕自己没办法面对后果，其实没什么是走不过的坎儿，阻挡你的只是懦弱和懒惰罢了
最终上苍眷顾，十三个人里，我复试第四，我，似乎就这样成为准研究生了

似乎一切都是最好的选择，苦尽甘来谱写胜利者的篇章
四月份，一场重感冒降临，带来的症状似乎比新冠更严重
头晕，肺部感染，从我肺里咳出比胶水还粘稠的土黄色粘液
第三天，我终于扛不住了，去医院对面的小诊所打了点滴，和医生提了一嘴我青霉素过敏，皮试结果似乎也是如此
对于一个小诊所来说，患者过敏死在自己诊所里是谁都不愿遇到的事，于是他们给我换了别的药
印象中口腔泛出浓浓的苦味，在吊完这一瓶之后，我遇到了我这一生中最严重的大脑宕机
完全没办法集中注意力，思维变得迟缓，眼睛无法聚焦
一开始还以为是太疲倦导致的，后面几天出现的症状让我感到恐惧
我开始出现头晕，易疲劳，肌肉控制僵硬等情况
我开始害怕了，不是害怕自己会因为生病而死去，而是害怕自己会失去自己最有价值的东西，灵魂和智识
因为这和我最恐惧的疾病，帕金森有几分相像
我艰难的在手机上敲下自己有生以来的第一篇遗书，尽管我没有任何财产能分配

我从学校骑车到东湖，一路上的美景并不能让我放松，而我担心我是不是可能会在某一次走神中失去生命
跑了两个医院，第一个医院的很好，先为我检查了肌肉的松弛度，表示没问题
在我表达了自己的顾虑之后，他给我开了MRI的检查，一查医院的排期已经去到了后日的凌晨三点，我咬牙同意了
交完钱不甘心的我前往了旁边的另一家三甲，幸运的是，今晚7:30就有空期，我不禁感慨南昌医疗资源的稀缺
医生给开了MRI和血管造影，交完钱，时间已然走到了5:20，饥肠辘辘的我走到旁边街道的火烧店，点了一份10块的火烧
看了一眼自己手机里的余额，咬了一口我觉得昂贵的火烧，嗯。。真TM难吃
思绪飘飞起来，我从小就是一个比较独立的人，遇到困难总想着自己解决，哪怕是大学的每一次生病去医院，我都没有通知父母向他们要过生活费之外的一分钱
可是这下我真的快要扛不住了，万一真的确诊，我怎么办，我年过六旬的父母怎么办，我该如何去面对这一切
心想着如果自己能不用担心检查的费用就好了，但是没人能帮我，口中嚼着发干发轫而又无味的火烧
回想起来，如果四下没人，我可能真的会捂面哭泣吧

回到医院，漫长的等待，脱下我的小包，走进寒冷的机房，盖上厚厚的被子，
医生叮嘱我脑部不可以移动，在我脑旁塞上泡沫限位器随后启动磁共振机
滋啦啦的声音在耳旁响起，仿佛审判一般，我没有紧张，没有不安，仿佛弥留之际一般平静
但急促的呼吸和能感受到的心跳仿佛还在提醒着我，我尚存这人界
当耳边急促的天堂建筑工地声归为平静，机器缓缓将我推出，
和医生最后和我说明日取报告时，我死死盯住他们，希望能扑捉点什么有用消息，很可惜，我失败了
第二天我请了半天假，拿着看不懂的报告，上面写着双侧脑白质多发缺血灶
重新交了22元的挂号费换来了另一位医生的一句“没什么问题”

不满足五个字诊断的我，开始找其他诊断，好在我是大学生，知道该如何学习新知识，在知网看了几篇论文和医学论坛解了更多的有关这种疾病的信息，我大致知道了这几点
1.这表示患者长期高血压导致脑部缺血，也可能是因为当时肺炎供氧不足加上输液导致的急性脑部缺血
这种脑损伤不可逆，多发老年人身上，可能会导致认知障碍，肌肉控制等
2.我的脑黑质正常，不是帕金森
3.我确实体质太差182/170就出现了长期高血压，应该要减肥，健康最重要
于是调整好心情边工作边锻炼身体，夜跑夜骑在两个月内瘦了20斤，身体状况也越来越好了

这也是我最快乐的大学时光之一，因为我和我对象在这段时间里几乎周周见面，我们逛完了南昌几乎所有的大商场，旅游景点，我们留下了很多美好的回忆
万寿宫，八一广场，万象城，卫东，紫荆夜市，我们有太多太多美好的快乐的回忆，我爱你洋宝！
两个灵魂相互了解，试探，相融，相爱，我们相互了解，规划未来，让我相信我能陪他走很远，很远
他给了我信心，在过往的恋情中扮演者伤害者角色的我有着不光彩的过去，他让我相信并证实我是一个有能力去爱，能给别人带来温暖的人
谢谢你，洋洋

总之，这是过去一年在我身上发生的一切，我最终还是捡起了这个我三年前用过的博客框架

linux系统的安装

前言

因为很多技术栈的技术都要求linux系统,安装linux系统得益于现在各大厂商对安装过程的简化,和图形化安装界面的优化已经做得非常好了,但是依旧有部分新手确实不太好入门,故写此文章记录自己的安装过程并且,提供一种安装教程供大家参考

step0:发行版的选择

linux目前用户数使用的比较多的是ubuntu和debian,使用apt管理包为首的linux发行版,这个时候又出现了许多的桌面环境例如KDE,xfce等,我个人和linux的创始者一样喜欢xfce的桌面环境,因为它功能足够,系统占用小,使用流畅(电脑太差了呜呜呜)
咳咳不多逼逼,这里我推荐的是xubuntu和linux mint xfce桌面版本

xubuntu:不太好看,但是性能强大的ubuntu发行版
xunbuntu

linuxmint:使用了xfce性能强大,更新好用的ubuntu内核发行版
linuxmint_XfceEdition

step1:进行下载并且进行安装准备

呜呼呼,终于来到第二部分了,点击下载就可以了,这里我们以linuxmint作为默认系统进行演示

您可以从Linux Mint官网https://linuxmint.com下载Linux Mint。

安装 Linux Mint 最简单的办法是用U盘安装。

在 Windows，Mac OS 或者其它的 Linux 发行版中建议使用Etcher

使用 Etcher 点击 Select image，然后选择您的ISO文件。
点击 Select drive，然后选择您的U盘。
点击 Flash!。

完成之后
现在您在U盘或DVD上有了一个可用于启动计算机的 Linux Mint 系统。
将U盘或DVD插入您的计算机。
重启计算机。

大部分BIOS有一个特殊的键，您可以按该键以选择启动设备。所有的BIOS都有一个特殊的键来进入BIOS设置屏幕，你可以在该设置屏幕中调整启动顺序。 对不同的BIOS，这个特殊键可能是 Escape， F1， F2， F8， F10， F11， F12，或者是 Delete。这些信息通常在启动时短暂地出现在屏幕上。

不同的品牌建议百度型号+bios,开机的时候按键盘上对应的按键即可

Linux Mint ISO 可以在 EFI 或 BIOS 模式中启动。在 EFI 模式中它会显示一个 grub 菜单。在 EFI 模式中它会显示一个 isolinux 菜单。

直接让他进入默认的livecd就可以进行安装了

step2:进行安装

Mint 实时会话
当你从u盘(或DVD)启动电脑时，Linux Mint启动了一个实时会话。它会自动以用户名为mint的用户登录，并向你展示有着安装程序的桌面：

在您的计算机上安装 Linux Mint:

双击光盘样式的快捷方式

选择您的语言。

连接到互联网。

如果您连接到互联网，请在方框中勾选以便安装多媒体编解码器。

选择一个安装类型。

如果Linux Mint是你想在这个电脑上运行的唯一系统并且硬盘上所有的数据都可以丢失，选择: Erase disk and install Linux Mint。

Linux Mint需要在一个分区上挂载根目录 / 。
Linux Mint操作系统（没有额外的软件和个人数据）大概需要15GB，所以给这个分区一个合适的大小（100GB或者更多）。
推荐您使用 ext4 。它是 Linux 中最流行的文件系统。

选择您的时区

选择您的键盘布局

请填入您的用户详情

你的名字可以是你的真实姓名，但这不是必须的。它只在本地使用，像屏幕保护程序和登录屏幕。
你的“用户名”是你登录的用户名，你的“主机名”是你的计算机在网络上的名字。
为了防止错误，只使用小写字符，没有标点符号或重音。
为了保护你的个人数据免受本地攻击（在你身边的人或者以防你的电脑被偷），勾选 Encrypt my home folder。
设置一个强密码。

当安装完成，点击 Restart Now。

重启完成之后您就拥有了一台linux系统的电脑啦!

echo命令

echo命令可以将命令行中的参数输出到标准输出（例如屏幕上）

date命令

运算符重载知识点及示例

0.自定义类的运算符重载概述(似乎没什么用)

C++中预定义的运算符的操作对象只能是基本数据类型。但实际上，对于
许多用户自定义类型（例如类），也需要类似的运算操作。这时就必须在
C++中重新定义这些运算符，赋予已有运算符新的功能，使它能够用于特
定数据类型，这就是运算符重载。运算符重载表现了C++的可扩展性。
• 运算符函数的定义与其他函数的定义类似，惟一的区别是运算符函数的函
数名是由关键字operator和其后要重载的运算符符号构成的。
• 运算符重载时要遵循的规则：
• （1）除了类属关系运算符“.”、成员指针运算符“.*”、作用域运算符“::”、
sizeof运算符和三目运算符“?:”等5种运算符以外，C++中的所有运算符都可
以重载。
• （2）重载运算符限制在C++语言中已有的运算符范围内的允许重载的运算符
之中，不能创建新的运算符。
• （3）运算符重载实质上是函数重载，因此编译程序对运算符重载的选择，遵
循函数重载的选择原则。
• （4）重载之后的运算符不能改变运算符的优先级和结合性，也不能改变运算
符操作数的个数及语法结构。
• （5）运算符重载不能改变该运算符用于基本数据类型变量的原有含义。它只
能和用户自定义类型的对象一起使用，或者用于用户自定义类型的对象和基
本数据类型的变量混合使用时。
• （6）运算符重载是针对新类型数据的实际需要对原有运算符进行的适当的改
造，重载的功能应当与原有功能相类似。
• 对于自定义类的运算符重载函数，可以将其定义为类的成员函数，也可以将
其定义为类的非成员函数，为了方便，非成员函数一般采用友元函数形式。

1.类成员函数形式的运算符重载

运算符重载的方法是定义一个重载运算符的函数。运算符函数重载一般有两
种形式：重载为类的成员函数和重载为类的非成员函数。

成员函数形式的运算符重载函数定义的一般形式为：

<返回类型说明符> operator<运算符符号>(<参数表>)
{
<函数体>
}

• 当运算符重载为类的成员函数时，函数的参数个数比原来的操作数要少一个
（后增、后减单目运算符除外）。因为成员函数用this指针隐式地访问了类的
一个对象，它就是运算符函数最左边的操作数。
• 调用成员函数运算符的形式如下：
• <对象名><运算符><参数> • 它等价于：
• <对象名>.operator<运算符>(<参数>)

类友元形式的运算符重载

由于友元函数可以访问类的私有成员和保护成员，为了方便，类非成员函数
形式的运算符重载函数一般采用友元函数。
• 运算符重载为类的友元函数，需要在类内进行声明。声明类的友元运算符重
载的形式为：
• friend <函数类型> operator<运算符>(<参数表>);
• 当运算符重载为类的友元函数时，由于没有隐含的this指针，因此操作数的个
数没有变化，所有的操作数都必须通过函数的形参进行传递，函数的参数与
操作数自左至右一一对应。
• 调用友元函数运算符的形式如下：
• <参数1><运算符><参数2>
• 它等价于：
• operator <运算符>(<参数1>,<参数2>)
• 【例2-18】利用友元运算符重载函数实现两个复数对象的加法计算。

• 在多数情况下，将运算符重载为类的成员函数和类的友元函数都是可以的，
采用何种形式，可参考下面的规则：
• （1）一般情况下，单目运算符最好重载为类的成员函数；双目运算符则最
好重载为类的友元函数。
• （2）以下一些双目运算符只能重载为类的成员函数：=、()、[]、->。 • （3）若一个运算符的操作需要修改对象的状态，选择重载为成员函数较好。
• （4）运算符所需的操作数（尤其是第一个操作数）希望有隐式类型转换，
则只能选用友元函数。
• （5）当运算符函数是一个成员函数时，最左边的操作数（或者只有最左边
的操作数）必须是运算符类的一个类对象（或者是对该类对象的引用）。如
果左边的操作数必须是一个不同类的对象，或者是一个内部类型的对象，该
运算符函数只能作为一个友元函数来实现。
• （6）当需要重载运算符具有可交换性时，选择重载为友元函数

2.类模版

2.1类模版的定义

定义:类模板可以用来描述一个与数据类型无关的类。类模板中类的
数据成员和成员函数的参数和返回值可以是任意的数据类型，
它描述了一组类的属性和行为，是一组类的统一描述，可以避
免类的重复定义。

定义类模板的一般格式为：
template <模板形参表>
class 类模板名
{
类体
};

template<typename 类型>
class 类名
{
类体(需要使用定义的类型)
}
template：定义模板的关键字，指明是函数或类模板。
模板形参表：用尖括号括起来。用来说明一个或多个类型形参和普通形参。多个模
板参数用“，”分隔。类型形参由关键字typename及其后面的标识符构成。该标
识符对应的实参可以是系统的基本数据类型，也可以是用户自定义的数据类型。
普通形参的说明方式为：<类型><普通形参名>

2.3类模版静态成员

类模板也允许有静态成员。静态成员将会是类模板实例化类的静态成员，
即对于一个类模板的每一个实例化类，它的所有对象共享类中的静态成员。

类模板的静态成员在定义类模板时不会被创建，在创建类模板的
实例化类后，在模板类中才会产生所有对象共享的静态成员。例如：

TA iobj1, iobj2;
TA dobj1, dobj2;

实例化模板类TA的两个对象iobj1和iobj2将共享TA
的静态成员int m_t；

实例化模板类TA的两个对象dobj1和dobj2将共享
TA的静态成员double m_t。

由于静态数据成员不属于任何对象，所以在创建对象时也不会为该
类的静态数据成员分配存储空间。所以，类设计者需要在类外对静态
数据成员进行定义。静态数据成员的定义形式如下：
<类型> <类名>::<静态数据成员名>[=<初值>];
例如：
int TA::m_t=100;
double TA::m_t=123.45;
